# JD 分析报告解读规则

## 报告 5 章节含义

| 章节 | 看什么 | 对用户的意义 |
|------|--------|------------|
| 一、数据概览 | 样本量、公司数、城市、实习/正式比例 | 判断市场需求量级和集中度 |
| 二、硬技能需求 TOP 榜 | TF-IDF/词频挖掘的技术关键词排行 | 知道该学什么、按权重排序 |
| 三、软技能与经验门槛 | 学历/经验分布 | 判断自己是否达标、差距在哪 |
| 四、行业赛道分析 | 招聘公司画像、大厂参与度 | 知道哪些公司在招、赛道热度 |
| 五、核心洞察总结 | 自动生成的建议 | 综合判断和行动指南 |

## 技能差距分析原则

1. 先看硬技能榜单 TOP 10 — 这些是必须掌握的
2. 再看学历/经验门槛 — 这是硬约束
3. 最后看软技能 — 锦上添花，但非决定性
4. 差距分析要具体到"技能名 vs 用户会/不会"

## 学习周期时长指导

| 时长 | 适合人群 | 内容范围 | 产出标准 |
|------|---------|---------|---------|

## 商业模块方法论

流量获取模块：通过技术博客、视频教程、开源贡献建立个人技术品牌，增加被招聘方发现的机会。

接单兼职模块：通过接单平台、咨询服务、模板化交付将技能快速变现，积累项目经验。

输出策略：商业模块是附加内容，不替代核心技术学习。时间紧张时应优先保证核心技术。

## 深度学习方法论：费曼 + 第一性原理

学习计划深度嵌入了两种核心方法论：

### 费曼学习法
- **每日**：学完知识点后，用大白话向小白讲清楚，打比方/举例子，卡住即盲区
- **每周**：所有概念串联成一个完整技术故事
- **产出**：费曼笔记就是内容资产，流量方案中可直接加工成图文/视频

### 第一性原理
- **阶段末**：5 层追问（问题本质 → 需求溯源 → 核心原理 → 边界认知 → 重构思考）
- **结业**：核心技术拆解文档（问题本质 → 核心原理 → 最小实现 → 设计取舍）
- **关键**：学框架学的是设计模式而非 API，去掉框架也要能实现核心逻辑
